# Briefly summarize the DriverPass project. Who was the client? What type of system did they want you to design?

The DriverPass project was made by the owner to help students pass the DMV driver's test due to a high rate of failures. He believed the high rate of failure was due to a lack of training, in-person and via courses, so created DriverPass to help fill that gap. He wanted a system that would allow students to create an account and schedule an in-person lesson online with 3 different packages to choose from. There would also be online courses, practice tests, and progress tracking for these. This system would need to be cloud-based and online with role-based user access for the students, secretary, IT admin, and the owner, and have updates from the DMV automatically mesh with the content.

# How did you interpret the user’s needs and implement them into your system design? Why is it so important to consider the user’s needs when designing?

The user's needs were communicated via the DriverPass transcript exclusively, so really the only way to know the user's needs was to hear what the owner wanted to create and go from there. Knowing the user's needs when designing is paramount because they will be the deciding factor on whether or not a project is successful, efficient, and worth using. Especially if you are attempting to make a profitable project.
